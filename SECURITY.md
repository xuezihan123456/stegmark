# Security Policy

## Supported Versions

This repository is currently pre-1.0 and evolves quickly.

Security fixes are applied to the latest branch state first.

## Reporting a Vulnerability

Please do not open a public issue for security-sensitive problems.

Instead, report the issue privately with:

- a short description of the impact
- reproduction steps
- affected files or commands
- any suggested mitigation

Until a dedicated security contact is configured, treat this repository as needing private maintainer coordination before disclosure.

## Scope

Please report issues involving:

- arbitrary file access
- unsafe shell execution paths
- dependency or model-loading trust boundaries
- malformed image inputs causing unsafe behavior
- release and benchmark automation supply-chain risks
