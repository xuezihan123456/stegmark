# StegMark v0.1 Implementation Plan

> **For agentic workers:** REQUIRED SUB-SKILL: Use superpowers:subagent-driven-development (recommended) or superpowers:executing-plans to implement this plan task-by-task. Steps use checkbox (`- [ ]`) syntax for tracking.

**Goal:** Build a pip-installable `stegmark` package with a working default invisible watermark engine, CLI, Python API, tests, and packaging metadata.

**Architecture:** Keep the repository aligned with the design document's layered shape, but implement only the `v0.1` slice. A built-in native engine provides guaranteed offline functionality, while an optional adapter layer preserves compatibility with future TrustMark-style integrations.

**Tech Stack:** Python 3.13, Click, Pillow, NumPy, pytest, hatchling, Rich, optional trustmark integration
