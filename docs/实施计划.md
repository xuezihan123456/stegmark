### 阶段 1.2: 引擎抽象层 + TrustMark 适配（3-4 天）

- [ ] **引擎抽象基类**
  ```python
  # src/stegmark/core/engine.py
  class WatermarkEngine(ABC):
      @abstractmethod
      def encode(self, image: np.ndarray, message: np.ndarray) -> np.ndarray: ...
      @abstractmethod
      def decode(self, image: np.ndarray) -> np.ndarray: ...
      @property
      @abstractmethod
      def payload_bits(self) -> int: ...
  ```

- [ ] **TrustMark 引擎适配器**
  - `src/stegmark/core/trustmark.py`
  - 下载 TrustMark ONNX 模型（encoder + decoder）
  - ONNX Runtime 推理封装
  - 输入预处理（resize → 256×256）+ 输出后处理（残差上采样）

- [ ] **模型下载管理器**
  - `src/stegmark/core/weights.py`

- [ ] **消息编解码器**

## v0.2 — "Own Engine"（自研 HiDDeN 模型）

**目标**: 训练自己的 HiDDeN 模型，提供除 TrustMark 外的第二个引擎选择。

### 阶段 2.1: 训练基础设施（3-4 天）

- [ ] **数据管道**
  - `src/stegmark/data/dataset.py`: WatermarkDataset (COCO 子集)
  - 数据下载脚本 `scripts/download_data.py`（COCO 10K 子集）
  - 数据增强: RandomCrop(128), RandomHorizontalFlip, ColorJitter
  - DataLoader 配置 (num_workers=4, pin_memory=True)

- [ ] **模型定义**
  - `src/stegmark/nn/hidden/encoder.py`: 4 层 ConvBNReLU + 消息拼接 + 2 层融合
  - `src/stegmark/nn/hidden/decoder.py`: 7 层 ConvBNReLU + GAP + Linear
  - `src/stegmark/nn/hidden/discriminator.py`: 3 层 ConvBNReLU + GAP + Linear

- [ ] **噪声层**
  - `src/stegmark/nn/hidden/noise_layers.py`
  - Identity, Dropout, Crop, Cropout, GaussianNoise, GaussianBlur
  - 可微 JPEG 近似（集成 DiffJPEG 代码，~200 行）
  - 组合噪声选择器（按权重随机选择）

- [ ] **损失函数**
  - `src/stegmark/training/losses.py`
  - ImageLoss: MSE + LPIPS (AlexNet)
  - MessageLoss: BCEWithLogitsLoss
  - AdversarialLoss: BCE

### 阶段 2.2: 训练流水线（3-4 天）

- [ ] **训练器**
  - `src/stegmark/training/trainer.py`
  - Adam 优化器 + CosineAnnealingWarmRestarts
  - AMP 混合精度支持
  - 课程学习调度（4 阶段噪声递增）
  - Tensorboard 日志
  - 检查点保存/恢复

- [ ] **训练脚本**
  - `scripts/train_hidden.py`
  - YAML 配置文件 `configs/hidden_v02.yaml`
  - 命令行: `python scripts/train_hidden.py --config configs/hidden_v02.yaml`

- [ ] **实际训练**
  - Phase 1 (Epoch 1-50, 无噪声): ~20 分钟
  - Phase 2 (Epoch 51-100, 空域): ~20 分钟
  - Phase 3 (Epoch 101-200, JPEG): ~40 分钟
  - Phase 4 (Epoch 201-300, 全噪声): ~40 分钟
  - **总计: ~2 小时**（128×128, batch=32, 4060 Ti）

- [ ] **训练评估**
  - 验证各 Phase 收敛指标
  - 生成质量报告（PSNR/SSIM/BER 表格）
  - 选择最佳模型检查点

### 阶段 2.3: ONNX 导出 + 引擎集成（2-3 天）

- [ ] **ONNX 导出**
  - `src/stegmark/training/export.py`
  - Encoder + Decoder 分别导出
  - 动态 batch size 支持
  - 导出后验证: ONNX 推理结果 ≈ PyTorch 推理结果

- [ ] **HiDDeN ONNX 引擎**
  - `src/stegmark/core/hidden.py`: HiDDeNEngine(WatermarkEngine)
  - ONNX Runtime 推理封装
  - 消息编解码集成（BCH(63,36)）

- [ ] **引擎切换机制**
  - `--engine hidden` CLI 参数
  - `StegMark(engine="hidden")` API 参数
  - `auto` 模式: 优先使用 TrustMark（质量更好），fallback 到 HiDDeN

- [ ] **测试 + 更新文档**
  - HiDDeN 引擎单元测试
  - README 更新（新增自研模型介绍）
  - CHANGELOG 更新

**v0.2 发布标准**:
1. HiDDeN 模型训练完成，PSNR > 38dB
2. ONNX 导出成功，推理结果一致
3. `--engine hidden` 可正常切换
4. 无攻击 BER < 0.005

## v0.3 — "Compression Proof"（抗 JPEG + benchmark）

**目标**: 自研模型达到可用的 JPEG 鲁棒性，提供完整 benchmark 工具。

### 阶段 3.1: 增强训练（4-5 天）

- [ ] **升级到 256×256**
  - 修改训练配置: 256×256, batch=16, AMP
  - 训练时间: ~8 小时

- [ ] **JPEG 鲁棒性专项训练**
  - DiffJPEG 噪声层权重提升到 40%
  - QF 范围扩大到 40-95
  - 增加 JPEG + Resize 组合噪声

- [ ] **消息容量升级**
  - 48 bits → 64 bits
  - BCH(63,36) → BCH(127,64,t=10)
  - 更新编解码器

- [ ] **多轮迭代训练**
  - 目标: JPEG Q=75 BER < 0.05, JPEG Q=50 BER < 0.10
  - 根据评估结果调整噪声权重和损失权重

### 阶段 3.2: Benchmark 套件（3-4 天）

- [ ] **攻击模拟器**
  - `src/stegmark/evaluation/attacks.py`
  - 12 种攻击实现: jpeg_q50/75/90, resize_0.5/0.75, crop_0.5/0.75,
    gaussian_blur_1/2, gaussian_noise_0.03, brightness_1.3, dropout_0.1

- [ ] **自动化评估框架**
  - `src/stegmark/evaluation/benchmark.py`
  - 自动对每种攻击: embed → attack → extract → 计算 BER/PSNR
  - 输出格式: table (rich) / csv / json / html

- [ ] **benchmark CLI 命令**
  - `stegmark benchmark photo.png -m "test" --attacks all -o ./report/`
  - 各引擎对比（TrustMark vs HiDDeN）

- [ ] **CI 质量门控**
  - `tests/test_benchmark.py`: 关键指标自动回归测试
  - `.github/workflows/benchmark.yml`: 每周运行 + 质量门控脚本

**v0.3 发布标准**:
1. JPEG Q=75 准确率 > 95%（BCH 纠错后）
2. JPEG Q=50 准确率 > 90%
3. benchmark 命令完整可用
4. 与 TrustMark 的对比表格

## v0.4 — "Crop & Scale"（StegaStamp 架构升级）

**目标**: 升级到 StegaStamp 架构，抵抗几何变换攻击。

### 阶段 4.1: StegaStamp 模型实现（5-7 天）

- [ ] **U-Net Encoder**
  - `src/stegmark/nn/stegastamp/unet_encoder.py`
  - 5 层下采样 + 5 层上采样 + Skip Connections
  - Secret 预处理: FC → reshape → upsample → 与 cover 拼接

- [ ] **STN Decoder**
  - `src/stegmark/nn/stegastamp/stn_decoder.py`
  - Localization Network → Grid Generator → Sampler
  - theta 初始化为单位仿射矩阵
  - 消息解码 CNN (10-12 层)

- [ ] **增强噪声层**
  - 透视变换（随机四角偏移 + grid_sample）
  - 颜色偏移（亮度/对比度/饱和度 ±20%）
  - 运动模糊
  - 组合攻击（2-3 种叠加）

### 阶段 4.2: 训练 + ONNX 导出（5-7 天）

- [ ] **训练配置**
  - 256×256, batch=16, AMP, 梯度累积 2
  - 训练时间: ~30-40 小时
  - 课程学习: 数字攻击 → 几何攻击 → 组合攻击

- [ ] **评估目标**
  - 裁剪 25% 后 BER < 0.10
  - 缩放 50% 后 BER < 0.08
  - JPEG Q=50 + 缩放 75% 组合 BER < 0.15

- [ ] **ONNX 导出**
  - STN 组件需要特殊处理（grid_sample 的 ONNX 兼容性）
  - 验证推理一致性

- [ ] **引擎集成**
  - `src/stegmark/core/stegastamp.py`: StegaStampEngine
  - 100 bits 容量
  - 更新 `auto` 引擎选择策略

### 阶段 4.3: 批量处理优化（2-3 天）

- [ ] **GPU 批量推理**
  - ONNX Runtime GPU 批量推理
  - 目标: > 90 img/s (1080p)

- [ ] **多进程 CPU 批量**
  - ProcessPoolExecutor + 进度回调
  - 内存安全（避免泄漏）

**v0.4 发布标准**:
1. 裁剪 25% 准确率 > 85%
2. 缩放 50% 准确率 > 85%
3. 批量 1000 张无内存泄漏
4. 三引擎对比 benchmark

## 开发规范

### Git 工作流

- `main` 分支: 始终可发布状态
- `dev` 分支: 开发主线
- `feat/xxx` 分支: 功能开发
- `fix/xxx` 分支: bug 修复
- PR 合并到 `dev`，发布时 `dev` 合并到 `main` + 打 tag

### Commit Message

Conventional Commits 格式:
```
feat: 添加批量处理功能
fix: 修复 JPEG 编码时的内存泄漏
docs: 更新 CLI 使用示例
test: 添加抗裁剪测试
chore: 升级 ruff 到 0.5
```
